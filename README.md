# CHIPS Portfolio Website

## Google Sheet CMS

The site can be updated from Google Sheets without editing the HTML by hand.

- Scheduled sync: every 30 minutes through GitHub Actions
- Manual sync: GitHub Actions > `Sync CMS from Google Sheet` > `Run workflow`
- Local sync: `powershell -ExecutionPolicy Bypass -File scripts/sync-cms-from-google-sheet.ps1`
- Styled workbook template: `docs/templates/chips-cms-template.xlsx`
- CMS docs: `docs/google-sheet-cms.md`

## Cloudflare Pages

Production is connected to Cloudflare Pages:

- URL: https://chips-website.pages.dev
- Source: `Yujin-0224/chips-website`
- Branch: `main`

## Submission Pages

Voice actors can use these Cloudflare Pages forms:

- `/profile-create.html`: submit profile information and an optional profile image
- `/profile-edit.html`: select an existing actor and update profile text or image
- `/audio-create.html`: upload an audio sample directly into R2
- `/login.html`: sign in with an approved actor/admin account
- `/signup-request.html`: request an actor account for operator approval

The upload, edit, and account functions require the Cloudflare Pages R2 binding `CHIPS_MEDIA` mapped to the `chips-media` bucket. Audio files are stored under `audio/{actor_id}/{sample_id}.{extension}`, submission metadata is stored under `submissions/`, and account data is stored under `auth/`.

The submission pages can be used without Google Sheets. The audio form loads the current actor list from `data/cms-data.json`, so users pick an existing actor instead of typing names or IDs by hand.

Submitted profiles and audio are reviewed from `/admin.html`. Approving a submission updates `cms/cms-data.json` in R2, and the public site reads that R2 CMS file before falling back to the repository copy.

Account signup requests are also reviewed from `/admin.html`. Approving a request creates an actor account tied to one `actor_id`, so that account can only edit its own profile and upload audio for itself. The admin page can also create actor or admin accounts directly when accessed with `ADMIN_TOKEN`.

Cloudflare Pages is a static host, so local-only routes such as `/api/drive-audio/:fileId` are not available there. In production, audio should come from Cloudflare R2 URLs generated by the sheet sync. While R2 credentials are not configured, the browser falls back to direct Google Drive download URLs.

## 실행

로컬 백엔드 프록시까지 포함해서 실행:

```powershell
npm start
```

사이트 주소:

```text
http://127.0.0.1:8000
```

VS Code에서 이 폴더를 열고 `index.html`을 브라우저로 열면 바로 확인할 수 있습니다.

권장 확장:

- Live Server

Live Server를 쓰면 수정 후 새로고침 없이 레이아웃을 확인하기 편합니다.

## 구성

- `index.html`: 화면 구조
- `styles.css`: 레이아웃과 CHIPS 컬러 스타일
- `script.js`: 성우 목록, 필터, 프로필 상세, 샘플 재생, 문의 메일 연결

## 다음에 교체할 부분

- 실제 CHIPS 로고 이미지가 생기면 `.cracker-logo` 영역을 이미지로 교체
- 메인 배경 이미지는 `assets/chips-hero.jpg`로 저장하면 자동 적용
- 실제 로딩에는 압축된 `assets/chips-hero-optimized.webp`를 사용합니다.
- 첫 화면 감자칩 이미지는 `assets/chips-chip.png`로 저장하면 자동 적용
- 프로필 이미지는 압축된 `assets/sample_profile-optimized.webp`를 사용합니다.
- 실제 성우 사진이 생기면 `script.js`의 actor 데이터에 이미지 경로 추가
- 실제 오디오 파일이 생기면 현재 브라우저 톤 미리듣기 대신 `<audio>` 파일 재생으로 교체
- 문의 수신 주소는 `script.js`의 `contact@chips-voice.com`을 실제 이메일로 변경

정식 서버에서 문의 메일을 자동 발송하려면 EmailJS, Formspree, 또는 별도 백엔드 API 연결이 필요합니다.
